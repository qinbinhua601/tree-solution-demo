# Headless Tree 分页接入 Prompt

```md
请在当前基于 `@headless-tree/core` / `@headless-tree/react` 的树项目中，实现“根节点本地渐进分页 + 触底自动加载”。

要求：
- 根节点首次仍可一次加载完整数据
- 首屏只展示前 N 个根节点
- 滚动到底部后继续展示下一批根节点
- 子节点保持现有懒加载和展开行为
- 新增、删除、拖拽进出根目录后，分页状态仍然正确

实现约束：
- 不要做服务端真分页，不要请求下一页接口
- 不要把分页逻辑直接散落在业务 tree hook 里
- 请拆成“通用分页能力”和“headless tree 适配层”两部分
- 继续使用底部哨兵 + `IntersectionObserver` 做触底加载

通用分页能力至少应负责：
- 完整集合
- 当前可见数量
- 当前可见数据
- `hasMore`
- `isLoadingMore`
- `loadMore`
- 整批重置
- 新增一项
- 删除一项或多项

headless tree 适配层至少应负责：
- 接根节点完整集合
- 输出当前应渲染的根节点
- 暴露根节点加载更多状态和方法
- 在根节点 CRUD / 拖拽变化后同步分页状态

组件层需要完成：
- 滚动容器
- 底部哨兵
- 触底自动加载
- 底部状态展示

验收标准：
- 首屏只展示前 10 个根节点
- 滚动到底部后会继续展示更多根节点
- 加载中有明确状态
- 全部展示完后有“没有更多”之类的结束状态
- 根节点为空时有空状态
- 新建根节点后可立即看到新增项
- 删除根节点后分页状态不乱
- 拖拽节点进出根目录后分页状态正确
- 子节点懒加载、展开、重命名、拖拽等现有能力不回归

完成后请说明：
- 通用能力和业务适配层分别放在哪里
- 为什么这次实现属于“本地渐进分页”而不是“服务端真分页”
- 还可以如何继续演进
```
